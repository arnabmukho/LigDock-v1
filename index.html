LigDock v1 - Automated Docking Pipeline

Introduction

LigDock v1 is a powerful and automated docking pipeline designed to perform docking simulations using AutoDock Vina. This tool helps researchers to prepare protein and ligand structures, run docking simulations, and generate the final docked poses with both the protein and ligand included. The tool integrates several scientific software packages to streamline the docking process, making it easier and more efficient.

Features

  • Automated protein and ligand preparation
  • Energy minimization using UCSF Chimera
  • Docking simulations with AutoDock Vina
  • Final output includes both protein and ligand
  • Comprehensive logging and error handling

Requirements

  • AutoDock Vina
  • MGLTools
  • Open Babel
  • UCSF Chimera
  • Python 3.x
  • Biopython

Cloning the Repository

You can clone the repository from GitHub using the following command:

git clone https://github.com/arnabmukho/LigDock-v1.git

Usage

1. Place your protein file (e.g., `protein.pdb`) in the same directory as the script.
2. Create a directory named `ligands` and place individual ligand SDF files in that directory.
3. Run the docking script:
   ```bash
   ./LigDock.sh
   ```

Detailed Explanation

Steps in the Docking Pipeline

The docking pipeline follows these steps:

  • Prepare the Protein: The protein structure is cleaned by removing specified heteroatoms. Hydrogens are added, and energy minimization is performed using UCSF Chimera. The minimized protein structure is prepared for docking using MGLTools.
  • Prepare the Ligands: Each ligand in SDF format is converted to PDBQT format using Open Babel. Hydrogens and Gasteiger charges are added to the ligand. The ligand is prepared for docking using MGLTools.
  • Run Docking Simulations: Binding site coordinates are extracted using a Python script. AutoDock Vina is used to perform docking simulations for each ligand. The protein and docked ligand are combined into a single PDBQT file.
  • Output: Docked poses are saved in the `docked_poses` directory. Log files are saved in the `logs` directory. A summary log file is generated with the docking scores.
